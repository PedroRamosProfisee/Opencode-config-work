## Go/no-go guidance

Use this recommendation model:

| Condition | Recommendation |
|---|---|
| Strong pain + reachable buyer + price signal | Build narrow MVP |
| Strong pain but weak evidence | Validate first |
| Interesting tech but unclear buyer | Reposition before build |
| Buyer exists but channel weak | Run channel tests before build |
| High build cost + weak monetization | Do not build yet |
| Personal/family value high + market unproven | Build personal-first MVP, validate commercialization separately |
| Personal/family value high + market weak | Keep as personal/internal tool; do not force commercial launch |

## Output of this step

End with:

```markdown
My recommendation: <clear recommendation>
Confidence: low/medium/high
Why: <1-3 bullets>
Smallest useful next step: <concrete action>
```

## Pre-Handoff Checklist

Before writing artifacts, verify:

- Decision ledger is internally consistent.
- Personal/commercial success criteria are separated when relevant.
- Public vs internal MVP scope is clear.
- Monetization timing is clear.
- Output path is confirmed.
- Focused vs full packet choice is explicit.
