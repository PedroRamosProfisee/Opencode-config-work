# Agent flow hierarchy rooted at MM-router

This document defines the canonical agent flow MM-router orchestrates.

MM-Router Phase 1: Research
- Hooked: mm-researcher
Phase 2: Investigate
- Hooked: mm-investigator
Phase 3: Handoff
- Hooked: multimodel-architect, mm-handoff-writer
Phase 4: Review
- Hooked: mm-reviewer
Phase 5: Implement
- Hooked: cc-implementor, fc-implementor, fb-implementor
Phase 6: Test
- Hooked: mm-scene-tester or mm-tester

Not-hooked: debug, local-implementor
